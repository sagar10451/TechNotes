## Core Concepts

**Region** = Cluster of data centres

**AZ (Availability Zone)** = Logical grouping of data centres

**Edge Location** = Independent of Region and AZ, to serve content close to end user, to reduce latency

## Choosing an AWS Region

Consider the following factors:
- Compliance
- Latency
- Service availability
- Pricing

## AWS Service Scope

AWS services can be categorized by scope:
- Global
- Regional
- Zonal
